WHAT_IS_THIS

          briefing
        

        

          

            Vitronitor is a React boilerplate that ships one codebase to four targets — web (PWA), iOS, Android, and Electron — and keeps every one of them working when the network drops.
          

          

            Most cross-platform starters give you exactly one of these: native-feel UX, real offline-first sync, or self-hosted update pipelines. Vitronitor ships all three, wired together as a working app you can clone and run today.
          

          

            It's built on Vite + React 19 + Tailwind v4, with ElectricSQL replicating Postgres into IndexedDB, TanStack DB for live queries and an offline mutation queue, Capacitor for iOS/Android, and Electron with system-tray + IPC. Reference backend is Hono + Supabase + S3-compatible storage — all swappable. MIT licensed.
          

        

      


      
      

        

          
          
VITAL_SIGNS

          core pillars
        

        
    
          
  • 
            
    
              ●
              OFFLINE_SYNC
            
    
            
    
              ElectricSQL replicates your Postgres tables into a local IndexedDB replica on every platform. Mutations queue in a Write-Ahead Log when offline, drain on reconnect, and survive crashes. TanStack DB sits on top with live queries, so the UI updates the moment a row changes — locally or remotely. The reference example is a notes table with full CRUD + RLS + soft-delete.
            
    
          
    • 
          
  • 
            
    
              ●
              OTA_PIPELINE
            
    
            
    
              Push JS-only updates without an app-store review or a forced restart. iOS uses self-hosted Capgo; Electron uses a custom shell auto-updater plus a renderer state machine. You own the full publish → sign → serve → verify loop. No Capgo Cloud, no EAS Update, no GitHub Releases dependency. Bundles ship to any S3-compatible bucket.
            
    
          
    • 
          
  • 
            
    
              ●
              NATIVE_SHELL
            
    
            
    
              Capacitor 8 with a custom-plugin pattern handles iOS and Android, including native network and auth bridges. Electron 41 ships with a system tray, deep-link handler, IPC layer, and safeStorage-backed credential storage. It is a real native shell, not a thin web wrapper — you get keyboard shortcuts, OS integration, and offline launch out of the box.
            
    
          
    • 
          
  • 
            
    
              ●
              BACKEND_AGNOSTIC
            
    
            
    
              The repo's reference server is Hono, but the client only assumes the HTTP+JSON contracts in docs/BACKEND_CONTRACTS.md. Implement them on Bun, Cloudflare Workers, Deno, Express, Fastify, Elysia, Next.js API routes, FastAPI, Rails, Go, Rust, .NET, PHP — anything. The auth provider (Supabase) and object store (S3-compatible) are also swappable.

BOOT_SEQUENCE

          copy · paste · run
        

        

          Prereqs: Node 22+, a Supabase project, an Electric source (Cloud or self-hosted), an S3-compatible bucket for OTA bundles. Caddy if you run the reference Hono backend.
        

        

          

            
            ~ /vitronitor
            
          


$ git clone https://github.com/simonflore/vitronitor
$ cd vitronitor
$ bash scripts/setup.sh
$ cp .env.example .env.local
$ npm install --legacy-peer-deps
$ bash scripts/setup-signing-key.sh
$ npm run dev
# Vite SPA on :5173. Vitals nominal.
